# Развёртывание публичного демо

Как запустить PDF Signer в режиме **stateless публичного демо** — версии, которой
можно пользоваться без регистрации, где **ничего из загруженного не сохраняется
на сервере**.

> Нужен обычный персистентный деплой? Это просто `docker compose up` — см.
> [README](../README.md#русский). Это руководство — только про публичное демо.

---

## Что такое демо-режим (и чего в нём нет)

В демо-режиме (`DEMO_MODE=1`) бэкенд **ничего не хранит**:

- **Удаление фона** подписи по-прежнему выполняется на сервере, но обработанный
  PNG возвращается инлайном (base64) и **на диск не пишется**.
- **Библиотека подписей** и **история подписаний** живут только в **браузере**
  посетителя (IndexedDB). При экспорте браузер отправляет пиксели подписей вместе
  с запросом; сервер собирает страницу в памяти и ничего не сохраняет.
- Подписанный файл создаётся и скачивается как обычно — но **ни копия на сервере,
  ни запись истории** не сохраняются.
- В интерфейсе показывается **баннер** о том, что данные остаются в браузере.

| | Обычный (`docker compose up`) | Демо (`-f docker-compose.demo.yml`) |
|---|---|---|
| Подписи на сервере | Да (`/data/signatures`) | **Нет** |
| Копия результата на сервере | Да (`/data/output`) | **Нет** |
| История подписаний | Сервер (`/data/history`) | **Только браузер (IndexedDB)** |
| Том данных смонтирован | Да | **Нет** |
| Изоляция посетителей | Общий `/data` | **Полная** (на сервере ничего нет) |
| Баннер | Нет | Да |

**Чего демо-режим НЕ даёт:** в нём **нет аутентификации / логина** — демо
намеренно открытое. Изоляция между посетителями обеспечивается тем, что сервер
просто ничего не хранит, а не учётными записями. Встроенного rate-limiting нет,
кроме анти-DoS-лимитов ниже; для публичного доступа добавьте его на reverse-proxy.

---

## Требования

- **Docker** с **Docker Compose v2.24.4 или новее** — оверрайд демо использует
  merge-тег `!reset`, чтобы убрать том данных, а старые версии Compose его
  игнорируют. Проверка: `docker compose version`.
- Хост (небольшого VPS достаточно) и, для публичного URL, домен + TLS (ниже).

## Быстрый старт

```bash
git clone https://github.com/TinaUma/PDF_Signer.git
cd PDF_Signer
docker compose -f docker-compose.yml -f docker-compose.demo.yml up --build -d
```

Приложение доступно на **http://localhost:8080**. Остановить:

```bash
docker compose -f docker-compose.yml -f docker-compose.demo.yml down
```

Нужны **оба** файла `-f`: базовый задаёт сервисы, файл демо добавляет
`DEMO_MODE=1` и убирает том данных.

---

## Как убедиться, что демо-режим включён

1. **Эндпоинт конфига** — должен вернуть `demo_mode: true`:

   ```bash
   curl http://localhost:8080/api/config
   # {"demo_mode": true, "version": "1.1.0"}
   ```

2. **Баннер** — в интерфейсе есть полоса «демо-режим — данные хранятся в браузере».

3. **Итоговый конфиг** — проверьте, что у бэкенда есть флаг и **нет** тома данных:

   ```bash
   docker compose -f docker-compose.yml -f docker-compose.demo.yml config
   # backend.environment содержит DEMO_MODE: "1"; у backend нет монтирования `volumes:`
   ```

4. **Пустая библиотека в свежем браузере** — откройте демо в новом приватном
   окне; библиотека подписей и история пусты (они привязаны к браузеру).

## Публичный хостинг за HTTPS (рекомендуется)

Контейнер фронтенда слушает **порт 8080 по обычному HTTP**. Для публичного адреса
завершайте TLS на reverse-proxy перед ним и **не** выставляйте бэкенд напрямую —
у бэкенда нет host-порта, он доступен только через внутренний nginx-proxy
фронтенда в сети Compose.

**Caddy** (автоматический HTTPS):

```caddy
demo.example.com {
    reverse_proxy localhost:8080
}
```

**nginx** (со своими сертификатами / certbot):

```nginx
server {
    listen 443 ssl;
    server_name demo.example.com;

    ssl_certificate     /etc/letsencrypt/live/demo.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/demo.example.com/privkey.pem;

    client_max_body_size 50m;   # соответствует лимиту загрузки 50 МБ

    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

> Референсное живое демо так и работает: **https://tinacodes.space**.

Если нужно, чтобы контейнер слушал только на loopback хоста (единственная точка
входа — reverse-proxy), привяжите публикуемый порт к `127.0.0.1` в небольшом
дополнительном оверрайде, напр. `ports: ["127.0.0.1:8080:8080"]` на сервисе
frontend.

## Обновление / передеплой

```bash
git pull
docker compose -f docker-compose.yml -f docker-compose.demo.yml up -d --build
```

Так как демо ничего не хранит на сервере, передеплой не требует миграций данных.
Библиотеки посетителей сохраняются в их браузерах между передеплоями.

## Безопасность и ресурсы

- **Аутентификации нет by design** — не держите в демо ничего чувствительного,
  оно открыто всем.
- **Анти-DoS-лимиты** (действуют в любом режиме): загрузки ≤ 50 МБ; площадь
  растеризации страницы ≤ ~64 Мп с защитой Pillow от decompression-bomb; ≤ 500
  страниц; инлайн-payload `signatures_data` ≤ 900 КБ (→ HTTP 413) и ≤ 100
  уникальных подписей. Для публичного адреса добавьте rate-limiting на
  reverse-proxy.
- **Хранилище браузера** — библиотека/история растут в собственном IndexedDB
  посетителя. Если хранилище переполнится, приложение покажет понятное сообщение
  «хранилище заполнено», а уже скачанный файл не пострадает.
- **Безопасность путей** — id инлайн-подписей валидируются как UUID; некорректный
  или неизвестный id пропускается, а не используется как путь.

---

## Диагностика

**Баннер не появляется / данные всё равно сохраняются на сервере.**
Демо-режим фактически не включён. Частые причины:

- Запущен обычный `docker compose up` или забыт **второй** файл `-f`. Всегда
  передавайте оба: `-f docker-compose.yml -f docker-compose.demo.yml`.
- **Docker Compose старше v2.24.4** — тег `!reset` игнорируется, и базовый том
  данных остаётся смонтированным. Обновите Compose.

Проверка:

```bash
# Должно показать DEMO_MODE: "1" и НЕ показать том `source: data` у backend:
docker compose -f docker-compose.yml -f docker-compose.demo.yml config

# Должно вернуть demo_mode true:
curl http://localhost:8080/api/config
```

**Библиотека подписей пуста после открытия на другом устройстве/в другом браузере.**
Это ожидаемо — в демо-режиме библиотека и история хранятся в браузере (IndexedDB),
поэтому не переносятся между устройствами и не переживают очистку данных сайта.

**Загрузки падают с 413.** Файл превышает 50 МБ или (в демо) суммарные данные
инлайн-подписей превышают 900 КБ — используйте меньше / меньшего размера подписи.
